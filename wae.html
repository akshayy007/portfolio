The goal of this assignment is to publish an accessible report on the web that identifies key accessibility issues and recommends accessibility improvements for the same. I evaluated the accessibility of the IUPUI's university library website. I used accessibility evaluation tools like WebAIM's Wave and AlumniOnline Free WCAG 2.0 and Section 508 Web Accessibility Check to identify the accessibility issues. Even though I found several accessibility features in the website, there were some components of the website that failed to meet the web accessibility standards. I was able to recommend some code-based as well as content-based improvements that were easy to implement. Implementing these accessibility improvements will help the website match the WCAG 2.1 guidelines for web accessibility.

Problem 01: Missing form labels

The label is missing from a form field where the site asks users to input search query.

Recommendation - Provide an appropriate label for the text field by defining the ‘aria-label’ property inside the input tag. In this case, the label for the text input would be the same as placeholder text – ‘Search for articles, books, and more.’

Recommended HTML code:

<input id="ebscohostsearchtext" class="ul-search-form" name="ebscohostsearchtext" type="text" aria-label=” Search for articles, books, and more” placeholder=" Search for articles, books, and more..."/>

Problem 03: Empty links

There is no text description of the link.

Recommendation - Add descriptive text for the link tag <a> that describes the purpose of the link.

Recommended HTML code:
